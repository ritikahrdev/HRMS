## ❓ LIKELY QUESTIONS (with answers)
- **"Is our data secure?"** → Role‑based access (employees see only their own data), passwords hashed, 5,300+ security tests including injection/access checks, all passing. Data is in a managed Postgres database (Supabase).
- **"What does it cost to run?"** → Free tiers today (hosting, database, email). If we outgrow them, the paid upgrades are a few dollars/month — still a fraction of a per‑seat HR subscription.
- **"What if it goes down / who maintains it?"** → It's our code, fully documented, deployed automatically from our repository. Any developer can maintain or extend it.
- **"Can it handle the whole company / more people?"** → Yes — it already holds the full roster; adding people is one screen or a bulk Excel import.
- **"Can we brand it / add X?"** → Yes — company name, logo, modules, leave policies, salary structure are all configurable, and new features can be added since we own the code.
- **"Why is it a little slow to open?"** → The free hosting tier sleeps when idle to stay free; a ~$7/month upgrade makes it instant 24/7. (Already mitigated — there's an automatic keep‑warm option.)

## 🛟 IF SOMETHING BREAKS
- **Blank/slow screen:** it's waking from sleep — wait ~10s or refresh once. (You warmed it up, so this is unlikely.)
- **A click doesn't respond:** refresh the page (Ctrl+Shift+R) and continue.
- **Stay calm and narrate** — "let me refresh that" — never apologise at length; keep the momentum.
